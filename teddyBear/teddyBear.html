프로젝트 결과

  • 기존 사이트 텍스트 보다 전반적으로 크게 하여 가독성을 높임
  • 디자인 요소에 어울리는 스크립트를 활용, 과하지 않은 움직임 구현
  • young한 디자인과 핑크 포인트 컬러를 사용하여 친근한 테디베어 사이트 구현
  • 브랜치 메인 섹션이 크다는 의견, 모든 페이지에 적용되어 있는 버튼 사이즈가 크다는 의견이 있었음

개인 평가

  • index페이지 스크롤시 폰트색상이 변경되는 부분을 맡았지만자바스크립트에 대한 이해가 부족하여 팀원의 도움을 받아서 완성

잘한 점

  • 퍼블리싱 가이드를 토대로 클래스값과 주석을 누구나 알아보기 쉽게 작성
  • 테디베어 일러스트를 제작하여 스크립트를 활용해 애니메이션 구현
  • index, info페이지 swiper 플러그인 활용하여 적용
  • Git organization 으로 협업 및 버전관리

배운 점

  • 중요한건 기본

퍼블리싱 가이드를 처음 작성 해보았는데 선생님께서 퍼블리싱 가이드는 아주 세세하게 작성해야한다고 가이드를 해주셨습니다.프로젝트를 진행해 팀원과 코드를 공유해보니 퍼블리싱 가이드가 있음에도다른 팀원의 클래스 값 때문에 코드 수정에서 시간을 소요 하였습니다.프로젝트를 또 진행한다면 퍼블리싱 가이드를 더 꼼꼼하고 세세하게기준을 지켜서 작성 해야겠다고 깨달았습니다.

  • 도움 받았던 스크립트를 복기해보자

scroll 이벤트 → 이벤트 함수에서 querySelector().offsetTop을사용하여 폰트 요소의 상단 위치를 가져옴 →querySelector().clientHeight - documentElement.clientHeight을 사용하여 폰트 요소 에서 뷰포트의 높이를 뺀 값을 계산 → scrollY를 사용하여 폰트요소를 스크롤 → true인 경우 css에 주었 던 스타일값을 사용하여 width값을 설정, max-width를 100%로 설정하여 최대너비 제한, 스크롤 시 색상 변경되게 설정 →폰트 요소로 스크롤 하지 않은 경우 스타일값의 너비를 0%로 설정

개인 리뷰

이번 팀 프로젝트는 제출일을 제외하면 10일간의 시간이 주어졌습니다.10일간 팀원들과 사이트를 리디자인하며 5개월간 배운 것을 최대한 이 사이트에 나타내야 했고완벽하게 구현하여 완성하는 것을 목표로 설정하였습니다.

팀 프로젝트가 두 번째라 그런지 의사소통에는 문제가 없었지만내가 작성하지 않은 팀원의 코드를 훑었을 때 가이드가 있음에도 알아보기 힘들거나 다시 한번 물어봐야 하는 경우가 생겼습니다.저 또한 그 과정을 겪고 다른 팀원이 번거롭지 않게 코드를 수정 하고 주석을 수정 하는 과정을 거쳤습니다.

git을 이용하여 수정이 잦다 보니 merge가 자주 일어나고 그 과정에서 제가 수정한 코드가 사라지는 일이 발생하였습니다. 코멘트에 더욱 상세하게 수정한 내용을 작성하고 겹치지 않게 작업하고 있는 곳 을 공유하여 같은 일이 반복되지 않게 하였습니다.

자바스크립트 이해도가 낮아 메인페이지에 맡은 스크립트 구현에 실패 하였지만,팀원의 도움으로 어떤 방식으로 구현할 수 있는지 알게 되었습니다.
